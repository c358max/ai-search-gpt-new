# 14. Docker 개발서버 배포

## 목적
- 클라우드 개발서버에 모델 비교용 앱을 Docker로 올린다.
- 모델별 포트와 인덱스/alias를 분리한 상태로 배포한다.

## 준비
- Docker Engine / Docker Compose Plugin 설치
- 개발 Elasticsearch 접속 정보 확보
- 서버 방화벽에서 `8091~8093` 오픈
- 모델 저장 디렉터리 준비: `/data/ai-search/models`

## 파일
- [`Dockerfile`](/Users/davidnam/Project/pulmuone/ai-search-gpt/Dockerfile)
- [`docker-compose.dev.yml`](/Users/davidnam/Project/pulmuone/ai-search-gpt/docker-compose.dev.yml)
- [`.env.dev.example`](/Users/davidnam/Project/pulmuone/ai-search-gpt/.env.dev.example)

## 1. 코드 반영
```bash
git pull
cp .env.dev.example .env.dev
```

모델 디렉터리를 먼저 만든다.
```bash
mkdir -p /data/ai-search/models
mkdir -p /data/ai-search/models/multilingual-e5-small-ko-v2
mkdir -p /data/ai-search/models/KURE-v1
mkdir -p /data/ai-search/models/bge-m3
```

서버에 모델 파일을 복사한다.
```bash
scp -r ./src/main/resources/model/multilingual-e5-small-ko-v2 <user>@<server>:/data/ai-search/models/
scp -r ./src/main/resources/model/KURE-v1 <user>@<server>:/data/ai-search/models/
scp -r ./src/main/resources/model/bge-m3 <user>@<server>:/data/ai-search/models/
```

`.env.dev`에서 아래 값을 먼저 수정한다.
- `AI_SEARCH_MODELS_ROOT`
- `AI_SEARCH_ES_URL`
- `AI_SEARCH_ES_USERNAME`
- `AI_SEARCH_ES_PASSWORD`

최초 배포는 재색인이 필요하므로 아래 값을 유지한다.
```dotenv
MODEL_EXTRA_PROFILES=,indexing-web,search-vector-only
```

이미 인덱스/alias가 준비된 뒤 웹만 다시 띄울 때는 이렇게 바꾼다.
```dotenv
MODEL_EXTRA_PROFILES=,search-vector-only
```

## 2. 이미지 빌드
```bash
docker build -t ai-search-gpt:dev .
```

## 3. 컨테이너 실행
```bash
docker compose --env-file .env.dev -f docker-compose.dev.yml up -d
```

기본 실행 대상은 `e5-small-ko-v2`, `kure-v1` 2개다.

`bge-m3`까지 같이 띄울 때만 optional profile을 추가한다.
```bash
docker compose --profile optional --env-file .env.dev -f docker-compose.dev.yml up -d
```

## 4. 상태 확인
```bash
docker compose --env-file .env.dev -f docker-compose.dev.yml ps
docker compose --env-file .env.dev -f docker-compose.dev.yml logs -f ai-search-kure-v1
```

브라우저 확인:
- `http://<server-host>:8091/model-compare.html`

API 확인:
- `http://<server-host>:8091/api/search/model-info`
- `http://<server-host>:8091/api/search?q=어린이%20간식&page=1&size=5`

## 5. 재배포
```bash
git pull
docker build -t ai-search-gpt:dev .
docker compose --env-file .env.dev -f docker-compose.dev.yml up -d
```

## 6. 중지
```bash
docker compose --env-file .env.dev -f docker-compose.dev.yml down
```

## 메모
- 같은 서버에 Elasticsearch가 이미 떠 있다면 `AI_SEARCH_ES_URL`을 그 서버 내부 접근 주소로 맞춘다.
- 모델은 이미지에 포함하지 않고 `/data/ai-search/models`에서 볼륨 마운트한다.
- 기본 compose는 `e5-small-ko-v2`, `kure-v1` 2개를 띄우고, `bge-m3`는 optional profile로 분리했다.
- `bge-m3`가 가장 무거우므로 메모리 압박 시 마지막에 추가하는 편이 안전하다.
