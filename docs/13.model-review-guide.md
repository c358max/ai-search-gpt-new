# 13. 모델 비교 검토 가이드

## 목적
- 기획자가 여러 한글 벡터 모델의 검색 결과를 직접 비교할 수 있는 상태를 만든다.
- 1차 목표는 운영 모델 1개를 선정하는 것이다.

## 모델/포트
| 모델 | 포트 | read alias |
|---|---:|---|
| `e5-small-ko-v2` | `8091` | `food-products-e5-small-ko-v2-read` |
| `kure-v1` | `8092` | `food-products-kure-v1-read` |
| `bge-m3` | `8093` | `food-products-bge-m3-read` |

## 실행 순서
1. 전체 모델 색인 + 웹 실행
```bash
./sh_bin/31_start_all_model_indexing_web.sh
```

2. 상태 확인
```bash
./sh_bin/35_check_model_status.sh
```

3. 대표 쿼리 비교
```bash
./sh_bin/34_compare_model_search_queries.sh
```

4. 필요 시 단건 비교
```bash
./sh_bin/33_compare_model_search.sh "어린이 간식"
```

## 대표 쿼리
- `어린이 간식`
- `건강한 간식`
- `생새우`
- `단백질 많은 간편식`
- `채식 위주의 건강식`

## 검토 기준
- 검색 의도와 맞는 상품이 상위 5개에 포함되는가
- 너무 무관한 상품이 상위에 섞이지 않는가
- 특정 카테고리로 과도하게 쏠리지 않는가
- 결과가 비거나 500 오류가 나지 않는가

## 브라우저 직접 확인
- `http://localhost:8091/api/search?q=어린이%20간식`
- `http://localhost:8092/api/search?q=어린이%20간식`
- `http://localhost:8093/api/search?q=어린이%20간식`
