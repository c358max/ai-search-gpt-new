# 15. 로컬 실행 체크리스트

<!-- DOC_STYLE_GUIDE -->
> 📌 <font color="#FFD700"><b>핵심</b></font>: 이 문서는 로컬에서 Elasticsearch부터 Spring Boot 서버 실행까지 성공시키는 순서를 체크리스트로 정리한 문서입니다.
> ⚠️ <font color="red"><b>주의</b></font>: 순서를 건너뛰면 `Pending`, `index_not_found_exception`, `all shards failed` 같은 오류가 발생할 수 있습니다.
> ✅ <font color="green"><b>권장</b></font>: 각 단계마다 "성공 확인"까지 보고 다음 단계로 넘어가세요.

## 대상
- 주니어 개발자
- 처음부터 로컬 환경을 다시 올려야 하는 개발자

## 목적
- Elasticsearch(ECK/K8s) 로컬 환경을 정상 기동한다.
- Nori 플러그인을 포함한 검색 환경을 준비한다.
- 모델 실행과 색인(alias 생성)까지 완료한다.
- 모델 비교 화면까지 정상 진입한다.

## 0. 먼저 알아둘 점
- `./gradlew bootRun`만 실행하면 검색 alias가 없어서 검색이 실패할 수 있습니다.
- 검색까지 하려면 `web`가 아니라 `indexing-web` 모드가 필요합니다.
- 모델 비교 화면은 `8091`, `8092`, `8093` 세 포트가 모두 떠 있어야 정상 동작합니다.
- Elasticsearch 상태는 최소 `green` 또는 준비 직후 `yellow` 수준이어야 합니다.

---

## 1. ECK 오퍼레이터 상태 확인

실행:
```bash
./sh_bin/check/01_check_eck_operator_status.sh
```

성공 확인:
- operator 관련 Pod가 `Running`

막히면:
- ECK가 아직 설치되지 않았다면 먼저 아래 실행
```bash
./sh_bin/setup/02_install_eck_operator.sh
```

---

## 2. Nori 포함 Elasticsearch 이미지 준비

최초 1회만 실행:
```bash
./sh_bin/setup/03_build_elasticsearch_nori_image.sh
```

성공 확인:
- `ai-search-es:8.13.4-nori` 이미지가 생성됨

막히면:
- 플러그인 다운로드 실패 시 `sh_bin/readme.md`에 있는 수동 zip 준비 절차 참고

---

## 3. Elasticsearch 클러스터 시작

실행:
```bash
./sh_bin/setup/05_start_elasticsearch_cluster_custom_image.sh
```

이 스크립트가 하는 일:
- 현재 프로젝트 경로를 기준으로 ES manifest 경로 자동 치환
- `ai-search` namespace 재생성
- Elasticsearch 클러스터 재기동

성공 확인:
- 마지막에 `[OK] Elasticsearch(custom image) is ready`

막히면:
- `Pending`이면 보통 스토리지나 이전 리소스 잔재 문제
- 다시 같은 스크립트를 실행하기 전에 기존 수동 `kubectl apply`는 섞지 않는 편이 안전

---

## 4. Elasticsearch 상태 확인

실행:
```bash
./sh_bin/check/03_check_elasticsearch_status.sh
```

성공 확인:
- `cluster health`가 `green`
- `unassigned_shards`가 `0`

추가 확인 포인트:
- `.synonyms-2` shard가 `STARTED` 상태면 동의어 시스템 인덱스도 정상

막히면:
- `red`이면 검색/색인보다 먼저 shard 문제를 해결해야 함
- `local port 9200 is already open`이 뜨면 기존 포트 점유 프로세스를 정리하고 다시 실행

---

## 5. Nori 플러그인 확인

실행:
```bash
./sh_bin/check/02_check_elasticsearch_nori_plugin.sh
```

성공 확인:
- `analysis-nori`
- `[OK] analysis-nori plugin is installed`

막히면:
- Pod가 `Running`이 아니면 먼저 3, 4번 단계 확인

---

## 6. truststore 준비

최초 1회 실행:
```bash
./sh_bin/setup/06_prepare_djl_truststore.sh
```

성공 확인:
- `~/.ai-cert/djl-truststore.p12` 생성

왜 필요한가:
- `indexing`, `indexing-web` 실행 시 모델 로딩에 필요할 수 있음

---

## 7. 단일 모델 색인 + 웹 실행

권장 예시:
```bash
AI_SEARCH_QUERY_EMBED_TIMEOUT_MILLIS=5000 ./sh_bin/model/03_run_model_indexing_web.sh e5-small-ko-v2
```

왜 `indexing-web`인가:
- 인덱스 생성
- 데이터 색인
- read alias 생성
- 웹 서버 유지

성공 확인:
- 로그에 `Index rollout complete`
- 로그에 `Indexing finished in indexing-web mode`

막히면:
- `index_not_found_exception`이면 보통 `web` 모드만 띄운 경우
- `Query embedding unavailable`가 자주 뜨면 `AI_SEARCH_QUERY_EMBED_TIMEOUT_MILLIS`를 `5000` 이상으로 증가

---

## 8. 모델 비교용 전체 실행

모델 비교 화면까지 보려면 아래 실행:
```bash
AI_SEARCH_QUERY_EMBED_TIMEOUT_MILLIS=5000 ./sh_bin/model/06_start_all_model_indexing_web.sh
```

이 스크립트가 띄우는 것:
- `e5-small-ko-v2` : `8091`
- `kure-v1` : `8092`
- `bge-m3` : `8093`

성공 확인:
- 3개 포트가 모두 떠 있음
- 각 모델 로그에 색인 완료 메시지 존재

막히면:
- 모델 하나만 띄우면 `model-compare.html`에서 일부 요청이 실패할 수 있음

---

## 9. 모델 비교 화면 접속

브라우저:
```text
http://localhost:8091/model-compare.html
```

성공 확인:
- 3개 모델 컬럼이 보임
- 검색어 입력 후 결과가 동시에 조회됨
- 결과 아래 공통 필터(인기순, 가격순 등)가 동작함

막히면:
- `요청 실패`가 뜨면 해당 모델 포트 서버가 살아 있는지 확인
- 평가 저장 실패가 뜨면 앱을 재시작해서 최신 백엔드 코드가 반영됐는지 확인

---

## 10. 자주 하는 실수

### 실수 1. `./gradlew bootRun`만 실행함
- 문제: 색인이 안 돼서 read alias가 없음
- 해결: `indexing-web` 모드로 실행

### 실수 2. 모델 하나만 띄우고 비교 화면 열기
- 문제: 비교 화면은 3개 모델 서버를 모두 호출함
- 해결: `./sh_bin/model/06_start_all_model_indexing_web.sh`

### 실수 3. Elasticsearch 상태를 확인하지 않고 앱부터 실행
- 문제: shard 문제 때문에 동의어/검색/색인이 연쇄 실패
- 해결: 항상 `03_check_elasticsearch_status.sh` 먼저 확인

### 실수 4. 임베딩 timeout이 너무 짧음
- 문제: 벡터 검색 대신 lexical fallback 발생
- 해결: `AI_SEARCH_QUERY_EMBED_TIMEOUT_MILLIS=5000`

---

## 11. 전체 모델 종료

모델 비교 작업을 끝냈으면 아래 스크립트로 전체 모델을 종료합니다.

실행:
```bash
./sh_bin/model/07_stop_all_models.sh
```

이 스크립트가 하는 일:
- `8091`, `8092`, `8093` 포트를 사용하는 모델 프로세스 종료
- pid 파일 정리

성공 확인:
- `[OK] stop request sent for all models`

언제 쓰나:
- 모델 비교 작업 종료 후
- 포트 충돌이 날 때
- 전체 모델을 다시 깨끗하게 띄우고 싶을 때

---

## 12. 가장 많이 쓰는 명령 모음

Elasticsearch 시작:
```bash
./sh_bin/setup/05_start_elasticsearch_cluster_custom_image.sh
```

Elasticsearch 상태 확인:
```bash
./sh_bin/check/03_check_elasticsearch_status.sh
```

Nori 플러그인 확인:
```bash
./sh_bin/check/02_check_elasticsearch_nori_plugin.sh
```

단일 모델 색인 (색인만 실행할 경우)
```bash
./sh_bin/model/04_run_model_indexing_only.sh e5-small-ko-v2
./sh_bin/model/04_run_model_indexing_only.sh kure-v1
./sh_bin/model/04_run_model_indexing_only.sh bge-m3
```

단일 모델 색인 + 웹:(최초 색인이 없는 경우)
```bash
./sh_bin/model/03_run_model_indexing_web.sh e5-small-ko-v2
./sh_bin/model/03_run_model_indexing_web.sh kure-v1
./sh_bin/model/03_run_model_indexing_web.sh bge-m3
```

단일 모델 웹: (색인데이터는 있고 웹서버만 재시작할 경우-전체모델 종료후 실행한다.)
```bash
./sh_bin/model/02_run_model_web.sh e5-small-ko-v2
./sh_bin/model/02_run_model_web.sh kure-v1
./sh_bin/model/02_run_model_web.sh bge-m3
```

전체 모델 비교용 실행:
```bash
./sh_bin/model/06_start_all_model_indexing_web.sh
```

전체 모델 종료:
```bash
./sh_bin/model/07_stop_all_models.sh
```

## 13. 한 줄 요약

로컬 성공 순서는 아래입니다.

1. ECK 확인
2. ES 시작
3. ES 상태 확인
4. Nori 확인
5. truststore 준비
6. `indexing-web`로 색인 + 웹 실행
7. 필요하면 3개 모델 전체 실행 후 비교 화면 접속
8. 작업이 끝나면 전체 모델 종료
