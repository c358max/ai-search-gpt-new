# 14-1. 사전 검증을 Spring 통합 테스트로 전환

## 목표
- 기존 `05_*` shell이 담당하던 사전 검증 절차를 Spring 통합 테스트로 옮긴다.
- shell에 흩어진 검증 로직을 테스트 코드로 모아 코드 이해도와 유지보수성을 높인다.
- 운영 절차와 검증 절차를 분리한다.

## 이전 shell 역할
- active userdict / stopword 파일 교체
- Pod 마운트 확인
- 인덱스 롤아웃
- `_analyze` 결과 검증
- with/without 시나리오 실행

## 전환 원칙
- 검증은 Spring 통합 테스트가 담당한다.
- shell은 운영 실행 경로만 최소한으로 남긴다.
- 검증용 shell은 삭제하고, 현재는 통합 테스트와 수동 절차를 사용한다.

## 패키지 구조 제안
- `src/test/java/com/example/aisearch/integration/dictionary/`
  - `DictionaryIntegrationTestBase`
  - `UserDictionaryAnalyzerIntegrationTest`
  - `StopwordAnalyzerIntegrationTest`

## 공통 support 책임
`DictionaryIntegrationTestBase`
- active 파일 백업/복구
- with/without fixture를 active 파일로 교체
- 인덱스 롤아웃 실행
- `_analyze` 호출
- 토큰 assertion helper 제공

## 테스트 책임
`UserDictionaryAnalyzerIntegrationTest`
- with-yalpi 적용 시 `얄피`가 단일 토큰으로 유지되는지 검증
- without-yalpi 적용 시 `얄피`가 분리되는지 검증

`StopwordAnalyzerIntegrationTest`
- with-stopword 적용 시 `식감`이 제거되는지 검증
- without-stopword 적용 시 `식감`이 유지되는지 검증

## 구현 기준
- 핵심 판단은 검색 결과가 아니라 `_analyze` 결과로 한다.
- vector 입력에는 stopword를 적용하지 않는다.
- 테스트는 `com.example.aisearch.integration...` 패키지 기준으로 `integrationTest` 태스크에서만 실행되게 둔다.

## 단계별 정리 계획
1. dictionary 통합 테스트/support 추가
2. 기존 shell 시나리오와 동일한 with/without 검증 확보
3. shell의 검증 책임 제거
4. 기존 검증용 shell 제거
5. 반영/검증은 통합 테스트와 문서 절차로 대체

## 현재 상태
- `05_1 ~ 05_5` 검증용 shell은 삭제됐다.
- dictionary 통합 테스트가 with/without 시나리오를 담당한다.

## 주의사항
- active 파일을 실제로 교체하므로 테스트 종료 후 원복이 반드시 필요하다.
- 사전류 테스트는 병렬 실행을 전제로 하지 않는다.
- `_analyze` 대상 analyzer는 shell과 같은 기준을 유지한다.
