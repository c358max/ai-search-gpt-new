# 12. 상용 전환 TODO List

## 목적
- 현재 프로젝트를 PoC/파일럿 단계에서 상용 쇼핑몰 검색 서비스 단계로 전환하기 위한 실행 항목을 정의한다.
- 원칙: 기능 추가보다 안정성/운영성/품질 검증을 우선한다.

## 현재 전제
- 현재는 샘플 데이터 기반 검색/색인/동의어/카테고리부스팅 기능이 구현된 상태다.
- 상용 기준의 성능, 가용성, 보안, 관측, 품질 평가 체계는 추가 구축이 필요하다.

## P0 (상용 오픈 전 필수)

### 1. 인프라 고가용성(HA) 구성
- Elasticsearch 운영 토폴로지 확정(멀티 노드, 역할 분리, 스토리지 정책).
- Kubernetes 배포에 리소스 요청/제한, anti-affinity, PDB, readiness/liveness 정비.
- 완료 기준:
  - 단일 노드 장애 상황에서 검색 API 가용성 유지.
  - 운영 리허설에서 장애 전환(runbook 포함) 검증 완료.

### 2. 보안/접근통제
- ES 계정/비밀번호 하드코딩 제거, Secret 관리 일원화.
- API 인증/인가(내부망 전용이라도 최소 서비스 인증) 적용.
- TLS(서비스 간 통신 포함) 적용 여부 확정 후 강제.
- 완료 기준:
  - 민감정보가 git/repo/application 기본값에 평문으로 남지 않음.
  - 운영 계정 권한이 최소권한(least privilege)으로 분리됨.

### 3. 검색 품질 기준선 수립  .. 안해도 될것 같음
- 오프라인 평가셋(대표 쿼리, 기대 카테고리/상품) 구축.
- 핵심 KPI 정의: CTR, 무결과율, 상위 N 정답률(NDCG/Recall@K 중 선택).
- 카테고리 부스팅/동의어 변경 시 회귀 평가 자동화.
- 완료 기준:
  - 배포 전후 품질 비교 리포트 자동 생성.
  - 품질 하락 시 배포 차단 게이트 동작.

### 4. 성능/부하 검증
- 목표 SLO 설정(p95/p99 지연, 에러율, 처리량).
- 트래픽 패턴 기반 부하 테스트(피크, 버스트, 장시간 soak) 수행.
- 병목 구간(임베딩, ES 쿼리, 네트워크) 계측 및 튜닝.
- 완료 기준:
  - 목표 트래픽에서 SLO 충족.
  - 장애 유도 테스트(ES 지연/일부 노드 다운)에서 서비스 열화 허용 범위 확인.

### 5. 관측성(Observability)
- 로그/메트릭/트레이스 표준화(요청 ID, 쿼리 타입, fallback 여부, latency).
- 대시보드/알람 구성(검색 오류율, 무결과율, ES 상태, 색인 실패율).
- 완료 기준:
  - 장애 시 원인 파악을 10분 이내에 시작 가능한 수준의 지표 확보.
  - 주요 알람 임계치와 온콜 대응 절차 문서화 완료.

### 6. 배포/롤백 자동화
- CI/CD 파이프라인에 테스트 + 품질게이트 + 배포승인 흐름 반영.
- 인덱스 롤아웃(alias swap) 실패 시 자동 롤백 정책 수립.
- 완료 기준:
  - 무중단 배포/롤백 시나리오를 스테이징에서 반복 검증.
  - 릴리즈 체크리스트와 책임자 승인 체계 확정.

## P1 (오픈 직후 안정화)

### 7. 데이터 파이프라인 운영화
- 실제 상품 마스터/재고/가격 동기화 정책 정의(증분/전체 재색인 기준).
- 색인 실패 재처리(재시도, DLQ 또는 실패 레코드 보관) 도입.
- 완료 기준:
  - 데이터 최신성 SLA 준수.
  - 실패 건 추적 및 재처리 절차 자동화.

### 8. 검색 기능 운영 도구 보강
- 동의어/카테고리부스팅 룰 변경 이력 관리(버전, 변경자, 변경사유).
- 운영자용 검증 API 또는 백오피스(샘플 쿼리 즉시 검증) 제공.
- 완료 기준:
  - 룰 변경이 감사 가능(audit trail)하며 롤백 가능.

### 9. 테스트 체계 강화
- 통합 테스트에 실환경 유사 데이터셋 확장.
- 계약 테스트(API 스키마/정렬 규칙) 및 회귀 테스트 케이스 추가.
- 완료 기준:
  - 주요 시나리오 회귀 테스트가 CI에서 안정적으로 동작.

## P2 (고도화)

### 10. 온라인 실험 체계(A/B Test)
- 랭킹 로직(부스팅, beta, threshold) 변경을 실험 단위로 분리.
- 실험 결과 기반 자동 리포트(CTR, 전환율, 구매기여) 구축.
- 완료 기준:
  - 검색 랭킹 변경이 감(感)이 아닌 데이터로 의사결정됨.

### 11. 비용 최적화
- ES 인덱스/샤드 전략 최적화, hot-warm 정책 검토.
- 임베딩 계산/저장 비용 분석 후 캐시 및 배치 전략 고도화.
- 완료 기준:
  - 월 비용 모니터링과 비용 대비 성능 지표 정착.
