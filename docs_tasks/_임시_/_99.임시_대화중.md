즉시 업데이트 가능

1. 20 문서에 현재 구현 전제가 단일 소스 + 단일 alias + 단일 설정이라는 점을 명시
   현재 코드는 IndexingRunner, IndexRolloutService, application.yml 모두 단일 mall 전제이므로, 문서의 출발점을 더 정확히 써둘 수 있습니다.
2. 20 문서의 색인 흐름 설명에 MallType만 추가하면 끝나는 구조가 아님을 반영
   mall별 source/mapper뿐 아니라 mall별 indexName/readAlias/synonymsSet까지 함께 전달하는 실행 컨텍스트가 필요하다는 점은 바로 문서화 가능합니다.
3. 20 문서의 검색 흐름 설명을 MallType Resolver -> 검색 컨텍스트 결정 -> 공통 검색 -> presenter 변환 순서로 구체화
   현재 문서보다 실제 구현 전환 경로가 더 명확해집니다.
4. 20 문서에 기존 타입 재사용 검토 항목 추가
   StandardSearchRequest, StandardSearchResult를 새로 만들지, 기존 ProductSearchRequest, SearchPageResult를 표준 타입으로 승격할지 검토가 필요하다는 문구는 바로 넣을 수 있습니다.
5. 20 문서에 테스트 범위를 좀 더 현실적으로 보강
   단위 테스트 외에 mall별 설정 선택, alias 분리, search presenter 변환 테스트가 필요하다는 점은 즉시 반영 가능합니다.
6. 20 문서에 이번 단계 범위 제외 섹션 추가
   예: 단일 인덱스 tenant 필드 방식, mall별 독립 ES 클러스터, Back Office UI는 현재 단계 제외.
7. 20_1 문서에 현재 한계가 스크립트만의 문제가 아니라 정적 매핑/정적 설정 구조에도 있다는 점 추가
   현재는 index-mapping.json, application.yml도 단일 mall 전제라는 점을 문서에 명시할 수 있습니다.
8. 20_1 문서에 전제 조건 섹션 추가
   mall별 alias, mall별 synonyms-set, mall별 mapping generation 없이는 사용자 사전 분리만으로 완결되지 않는다는 점을 선행조건으로 정리할 수 있습니다.
9. 20_1 문서의 재시작 전략 설명에 논리적 영향 범위와 인프라 적용 범위를 구분해서 서술
   사전은 mall별로 분리 가능하지만, ES 재시작 영향은 클러스터 공통일 수 있다는 점은 바로 문서화 가능합니다.
10. 20_1 문서의 검증 항목을 _analyze 중심에서 검색 회귀 검증 포함으로 보강
    대표 쿼리, 기대 토큰, 기대 상품 확인이 필요하다는 수준까지는 즉시 업데이트 가능합니다.
11. 20_1 문서에 이번 단계 범위 제외 섹션 추가
    예: 무중단 사전 hot reload, Back Office 승인 UI, mall별 ES 물리 분리 등은 제외.

논의/결정 필요
12. MallType 전달 방식 결정
    URL path, header, host, 토큰 클레임 중 무엇을 1차 표준으로 할지 정해야 합니다.

13. 표준 검색 모델 전략 결정
    StandardSearchRequest/Result를 새로 만들지, 기존 ProductSearchRequest, SearchPageResult를 공통 표준 모델로 재해석할지 결정이 필요합니다.
14. mall별 운영 설정 주입 방식 결정
    application.yml을 mall map 구조로 바꿀지, mall별 profile/env 조합으로 갈지 정해야 합니다.
15. 인덱스 롤아웃 실행 컨텍스트 설계 결정
    MallType만 받을지, mallId + indexName + readAlias + synonymsSet + dictionaryPath를 묶은 MallContext를 둘지 결정해야 합니다.
16. 인덱스 분리 전략 결정
    물리 인덱스 완전 분리와 단일 인덱스 + tenant 필드 중 어느 방식을 채택할지 정해야 합니다.
17. mall별 매핑 생성 방식 결정
    index-mapping.json을 mall마다 따로 둘지, 하나의 템플릿에 placeholder를 더 추가할지 결정이 필요합니다.
18. mall별 analyzer/tokenizer 네이밍 규칙 결정
    analyzer 이름을 공통으로 유지하고 경로만 바꿀지, analyzer 자체도 mall별 이름으로 분리할지 정해야 합니다.
19. 사용자 사전 ConfigMap 전략 결정
    mall별 개별 ConfigMap으로 갈지, 단일 ConfigMap 다중 key로 갈지 최종 결정이 필요합니다.
20. 사용자 사전 반영 시 ES 재시작 정책 결정
    전체 restart 유지, rolling restart, skip-restart 운영 옵션 중 1차 운영 표준을 정해야 합니다.
21. mall별 부분 롤아웃 방식 결정
    사전 변경 시 정말 해당 mall 인덱스만 재생성/alias 전환할지, 전체 재색인을 유지할지 결정해야 합니다.
22. 검색 응답 DTO 전략 결정
    mall마다 완전 별도 DTO/presenter로 갈지, 공통 DTO + mall별 확장 필드 방식으로 갈지 정해야 합니다.
23. mall별 원천 상품 표준 스키마 범위 결정
    StandardProductDocument에 무엇을 필수 공통 필드로 둘지, 확장 필드를 어디까지 허용할지 결정이 필요합니다.
24. 운영 권한 및 배포 절차 결정
    동의어/사용자 사전/카테고리 부스팅 변경 권한, 승인 절차, 배포 책임 주체를 정해야 합니다.
